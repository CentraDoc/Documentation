PieceInfo

The PieceInfo dictionary/stream is for private data.

The PieceInfo calls hide some of the details of stashing private data in the pdf document, by document and by page.

Restrictions:

  • the key must be a valid pdf name, starting with '/'
  • pieceinfo streams cannot be created while a page is being written, only after (the interrupt & continue stuff isn't plugged in here, it can be added if needed)
  • the void * can be cast to PdfDictionary * and used to store more structured data, but only before CreatePieceInfo is called; once the stream is created, no other dictionary keys can be added or modified - behavior is undefined (currently keys are lost)
  • Add will replace any existing key!

The /LastModified stuff seems a little slippery - a GetPieceInfo dictionary can be modified without the LastModified stuff getting updated. Probably not a big deal.

Detailed Description

Pdf High Level interface.

Within PdfDoc: PDF Document Package, the Pdf API presents the simpler high level API for reading and writing PDF files. Most of the actual implementation is in pdfdoc/pdfdoc.h and the core of the output and update is implemented in PdfUpd: PDF Document Create and Update Package, but Pdf presents a simplified approach to both subsystems.
