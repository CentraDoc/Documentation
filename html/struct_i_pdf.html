Fonts

There are several approaches to creating fonts:

  • Use one of the 14 Standard Type 1 PDF fonts
  • Embed an existing external font, typically TrueType
  • Create a Type 3 font on-the-fly, either stencil or fully colored. This approach uses a callback function.
  • Create a Type 1 font on-the-fly, one of several ways:
    • Create a Type1 definition using T1Gen, then create the font from that.
    • Use a callback function all-at-once, similar to Type 3 this uses the callback to populate the T1Gen object.
    • Use the callback but for each glyph one-at-a-time

PieceInfo

The PieceInfo dictionary/stream is for private data.

The PieceInfo calls hide some of the details of stashing private data in the pdf document, by document and by page. Restrictions:

  • the key must be a valid pdf name, starting with '/'
  • pieceinfo streams cannot be created while a page is being written, only after (the interrupt & continue stuff isn't plugged in here, it can be added if needed)
  • the void * can be cast to PdfDictionary * and used to store more structured data, but only before CreatePieceInfo is called = 0; once the stream is created, no other dictionary keys can be added or modified - behavior is undefined (currently keys are lost)
  • Add will replace any existing key! The /LastModified stuff seems a little slippery - a GetPieceInfo dictionary can be modified without the LastModified stuff getting updated. Probably not a big deal.
